# Rejection Pattern Library

Common failure modes and their correction prompts. Use these to identify and fix problems without starting over.

---

## How to Use

1. **Diagnose:** Compare failing output against patterns below
2. **Match:** Find the pattern that best describes the failure
3. **Correct:** Use the correction prompt to guide revision
4. **Verify:** Re-check against the original requirements

---

## Output Quality Patterns

### Pattern 1: Generic AI Voice

**Symptoms:**
- Phrases like "delve into," "tapestry of," "landscape of"
- Overuse of em dashes and compound sentences
- Abstract nouns instead of concrete specifics
- Hedging language ("may," "might," "could potentially")

**Detection:** Read aloud. If it sounds like a LinkedIn thought leader, this is the problem.

**Correction Prompt:**
> Rewrite with concrete specifics. Remove all hedging. Use short sentences. Say exactly what you mean, no more, no less. If you do not have specific information, state that directly rather than being vague.

---

### Pattern 2: Format Compliance Without Substance

**Symptoms:**
- All sections present but content is thin
- Bullet points that restate the heading
- Placeholders not replaced, or replaced with generic filler
- Audit checks pass but output is useless

**Detection:** Check if removing the template leaves any meaningful content.

**Correction Prompt:**
> The format is correct but the content is empty. For each section, answer: what specific decision or insight goes here? If you cannot answer, remove the section. Prefer incomplete-but-real over complete-but-empty.

---

### Pattern 3: Scope Drift

**Symptoms:**
- Output addresses topics not in the original scope
- Creative additions that were not requested
- Solving problems the user does not have
- Over-engineering simple requests

**Detection:** Compare output against the scope defined in Stage 01. Any content not traceable to scope is drift.

**Correction Prompt:**
> The output includes work outside the defined scope. Remove everything not explicitly requested or logically necessary for the requested output. Add nothing "extra" or "helpful" unless the user asked for it.

---

### Pattern 4: Context Reference Failure

**Symptoms:**
- Output contradicts reference files
- Ignoring brand voice, design system, or conventions
- Treating the task as starting from scratch
- Not using loaded context in the output

**Detection:** Check if output would be different if reference files were never loaded.

**Correction Prompt:**
> Re-read the loaded reference files. The output must reflect those constraints. List 3 specific rules or patterns from the references that apply to this output, then revise to follow them.

---

### Pattern 5: Shallow Synthesis

**Symptoms:**
- Findings listed but not connected
- No contradictions surfaced
- No conclusions drawn
- "Here is what I found" without "here is what it means"

**Detection:** If synthesis output could be generated by concatenating discovery outputs, it is too shallow.

**Correction Prompt:**
> Connect the findings. Where do sources agree? Where do they contradict? What pattern emerges across all sources? State 2-3 conclusions that are not explicitly in any single source but follow from combining them.

## Process Adherence Patterns

### Pattern 6: Skipped Steps

**Symptoms:**
- Output produced without following process steps in order
- Checkpoints bypassed
- Audit section not run
- Shortcuts taken "for efficiency"

**Detection:** Trace output back to process steps. Any step without a trace was skipped.

**Correction Prompt:**
> Follow the process steps exactly as written. Do not skip or combine steps. After each step, state what was done before moving to the next.

---

### Pattern 7: Premature Optimization

**Symptoms:**
- Solving problems that have not occurred yet
- Building abstractions for single-use code
- Adding configuration options nobody asked for
- Future-proofing instead of shipping

**Detection:** If any part of the output addresses "what if we need to..." without an actual requirement, this is the problem.

**Correction Prompt:**
> Remove everything not needed for the current requirement. Solve today's problem today. When a real need emerges, solve it then. Complexity without a current use case is waste.

---

### Pattern 8: Checkpoint Bypass

**Symptoms:**
- Agent proceeded past checkpoint without human input
- Assumed default choice instead of asking
- Presented options then immediately chose one
- "I'll proceed with X" without waiting

**Detection:** Any checkpoint in the process should have a corresponding human message in the conversation.

**Correction Prompt:**
> Stop at checkpoints. Present options. Wait for human direction. Do not choose on the user's behalf. If the user is unavailable, pause and state that you are waiting for input.

---

## Context Management Patterns

### Pattern 9: Context Window Waste

**Symptoms:**
- Loading entire files when sections were specified
- Reading outputs from unrelated stages
- Loading reference files not needed for current task
- Loading the same file multiple times

**Detection:** Compare what was loaded against the What to Load table in CONTEXT.md.

**Correction Prompt:**
> Check the What to Load table. Load only what is specified. If a file is not in the table, do not load it. If a section is specified, read only that section.

---

### Pattern 10: Session Memory Loss

**Symptoms:**
- Re-asking questions answered earlier in conversation
- Repeating analysis already completed
- Losing track of what stage is active
- Forgetting decisions made in previous steps

**Detection:** If the agent asks a question that was answered in the current session, this is the problem.

**Correction Prompt:**
> Read PROGRESS.md if it exists. Check the conversation history for previous decisions on this topic. State what you already know before asking or doing anything. Do not restart work that is in progress.

## Quick Reference

| Pattern | Category | Quick Fix |
|---------|----------|-----------|
| Generic AI Voice | Quality | Rewrite with specifics, no hedging |
| Format Without Substance | Quality | Answer what goes in each section |
| Scope Drift | Quality | Remove everything not in scope |
| Context Reference Failure | Quality | Apply loaded constraints explicitly |
| Shallow Synthesis | Quality | Connect findings, draw conclusions |
| Skipped Steps | Process | Follow steps exactly, in order |
| Premature Optimization | Process | Remove unused complexity |
| Checkpoint Bypass | Process | Stop and wait for input |
| Context Window Waste | Context | Load only what is specified |
| Session Memory Loss | Context | Read PROGRESS.md, check history |
